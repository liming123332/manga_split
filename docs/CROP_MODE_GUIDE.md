# 提取范围功能说明

## 概述

浮浮酱为主人添加了**可调节的提取范围**功能喵～

现在可以选择提取：
- **仅人脸** (face) - 只提取人脸区域
- **上半身** (upper_body) - 提取头部到胸部/腰部区域（推荐 ⭐）
- **全身** (full_body) - 尽可能提取完整人物

## 使用方法

### 方法 1：GUI 模式（推荐）

1. 启动 GUI：
   ```bash
   python gui.py
   ```

2. 在配置选项中找到"**提取范围**"：
   - ○ 仅人脸
   - ⦿ **上半身 (推荐)** ← 默认推荐
   - ○ 全身

3. 选择想要的提取范围
4. 点击"开始处理"

### 方法 2：配置文件

编辑 `config.yaml`：

```yaml
detection:
  crop_mode: "upper_body"   # 裁剪模式
  crop_padding_ratio: 0.5   # 额外边距比例
```

**可选值：**
- `"face"` - 仅人脸
- `"upper_body"` - 上半身（推荐）
- `"full_body"` - 全身

## 各模式详解

### 1. 仅人脸 (face)

**特点：**
- 裁剪范围最小
- 专注于面部特征
- 适合训练人脸识别模型

**提取逻辑：**
- YOLO: 使用检测框 + 20px 边距
- SAM: 裁剪人物上半部的中心 60%

**适用场景：**
- ✅ 人脸识别训练
- ✅ 表情分析
- ✅ 需要最小背景的场景

**示例输出：**
```
仅包含头部和颈部区域
```

### 2. 上半身 (upper_body) ⭐ 推荐

**特点：**
- 包含头部到胸/腰部
- 保留更多角色特征
- 平衡文件大小和内容完整性

**提取逻辑：**
- YOLO:
  - 向上扩展 0.5 倍边距
  - 向下扩展 2.5 倍人脸高度
  - 左右扩展人脸宽度的 50%

- SAM:
  - 使用人物的上 70%
  - 添加适度边距

**适用场景：**
- ✅ LoRA 训练（推荐）
- ✅ 角色风格学习
- ✅ 需要服装/发型特征
- ✅ **大多数二次元图像处理**

**示例输出：**
```
包含头部、肩膀、部分胸部/手臂
```

### 3. 全身 (full_body)

**特点：**
- 尽可能包含完整人物
- 保留全身服装和姿势
- 文件最大

**提取逻辑：**
- YOLO:
  - 向上扩展 0.5 倍人脸高度
  - 向下扩展 4 倍人脸高度
  - 左右扩展人脸宽度的 75%

- SAM:
  - 使用完整的人物检测区域

**适用场景：**
- ✅ 全身角色建模
- ✅ 服装/姿势学习
- ✅ 人物完整构图分析

**注意事项：**
- ⚠️ 可能包含大量背景
- ⚠️ 文件较大
- ⚠️ 如果人物不完整，效果可能不理想

**示例输出：**
```
包含从头到脚的完整人物（如果图像中有）
```

## 参数调优

### crop_padding_ratio

控制额外边距的大小（相对于人脸尺寸）：

```yaml
detection:
  crop_mode: "upper_body"
  crop_padding_ratio: 0.5    # 默认值，推荐
```

**推荐值：**
- `0.3` - 紧凑裁剪
- `0.5` - 标准边距（推荐）⭐
- `0.8` - 宽松边距
- `1.0` - 更大边距

## 常见问题

### Q1: 上半身模式可能截断人物的下半身吗？

**A:** 是的，这是预期行为。"upper_body" 专注于上半部分，适合大多数二次元图像。如果需要完整人物，请使用 "full_body"。

### Q2: 全身模式会检测到完整人物吗？

**A:** 取决于原图：
- 如果原图是全身立绘 → 可以提取完整人物
- 如果原图只是半身像 → 只能提取现有部分
- YOLO 只检测人脸，基于人脸位置推断身体范围

### Q3: 可以自定义扩展范围吗？

**A:** 可以调整 `crop_padding_ratio` 参数：
- 增大值 → 扩展更多范围
- 减小值 → 裁剪更紧凑

### Q4: SAM 和 YOLO 的提取范围一样吗？

**A:** 逻辑类似，但实现不同：
- **YOLO**: 基于人脸边界框扩展
- **SAM**: 先检测完整人物，再根据模式裁剪

SAM 通常能更好地判断人物边界。

### Q5: 提取范围是否会影响检测速度？

**A:** 不会。检测速度相同，只是输出图像的尺寸不同。

## 技术细节

### YOLO 扩展逻辑

```python
if crop_mode == "upper_body":
    expand_top = int(padding * 0.5)
    expand_bottom = int(h * 2.5)  # 关键：向下扩展 2.5 倍
    expand_left = int(w * crop_padding_ratio)
    expand_right = int(w * crop_padding_ratio)
```

### SAM 扩展逻辑

```python
if crop_mode == "upper_body":
    upper_body_ratio = 0.7  # 使用人物的上 70%
    upper_height = int(h * upper_body_ratio)
    # 添加适度边距
```
